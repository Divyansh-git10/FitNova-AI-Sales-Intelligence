# docs/screenshots/

Every image here is generated by `scripts/capture_screenshots.py` or
`scripts/render_dashboard_previews.py` against a real, freshly seeded
database — nothing in this folder is hand-typed or invented output.

## CLI (`cli_*.svg`)

Real Rich terminal recordings (`Console(record=True).save_svg()`) of
actual `fitnova` command runs: `doctor`, `status`, `ingest`, `analyze`,
`benchmark`. These are exact captures — open any of them in a browser to
see precisely what running that command produces.

## API (`api_walkthrough.svg`)

A real Rich terminal recording of real HTTP requests made against the
actual FastAPI ASGI app (`fitnova.api.main:create_app()`) via
`fastapi.testclient.TestClient` — the same app object `uvicorn
fitnova.api.main:app` serves. Response bodies shown are genuine JSON
returned by the app for a freshly seeded demo database.

## Dashboard (`dashboard_*_preview.svg`)

**These two are not literal browser screenshots.** This build environment
has no internet access to install a headless browser (Playwright needs to
download a ~150-300MB Chromium binary, which isn't reachable here), so a
pixel-exact capture of the Streamlit UI isn't obtainable in this sandbox.

What's here instead: `scripts/render_dashboard_previews.py` queries the
real `fitnova.db.repository` functions — the exact same functions the
actual dashboard pages call — against a freshly seeded demo database, and
lays the real numbers out in hand-drawn SVG that approximates each page's
structure. The data is 100% real; the pixels are not Streamlit's own CSS.
Every file says so in its own header text, too.

**To capture the real thing**, it takes about 30 seconds once you've
followed the setup guide:

```bash
python -m scripts.seed_demo_data
fitnova dashboard
# open http://localhost:8501, click through the pages, screenshot away
```

`docs/DEMO_VIDEO_SCRIPT.md` Scene 6 lists every page worth capturing, in
the order that tells the clearest story.

## Regenerating everything in this folder

```bash
python -m scripts.capture_screenshots
python -m scripts.render_dashboard_previews
```

Both scripts are hermetic — they build and seed an isolated temporary
database, so re-running them never touches your real `data/fitnova.db`.
